71 /// Compares the rows of one table across two databases.
72 ///
73 /// Both @p a and @p b are expected to contain a table identical to @p tableSchema —
74 /// callers should run the schema diff first and only invoke this for tables that match.
75 /// Pairs rows by primary key (using the columns listed in @c tableSchema.primaryKeys).
76 /// Tables without a primary key are skipped: the returned @ref TableDataDiff has an empty
77 /// @c rows vector and @ref TableDataDiff::skipReason set.
78 ///
79 /// All column values are compared as their ODBC string representation (the same coercion
80 /// used by `dbtool exec`). This may produce false positives for floating-point or binary
81 /// columns whose textual encoding differs across drivers.
82 ///
83 /// @param a Connection to the left-hand database (already connected).
84 /// @param b Connection to the right-hand database (already connected).
85 /// @param tableSchema Schema of the table; columns and primary keys are read from it.
86 /// @param maxRows Maximum number of rows to scan per side. 0 means unlimited.
87 /// @param onProgress Optional callback fired ~2 Hz with current scan counters.
88 LIGHTWEIGHT_API TableDataDiff DiffTableData(SqlConnection& a,
89 SqlConnection& b,
90 Table const& tableSchema,
91 std::size_t maxRows = 0,
92 DiffProgressCallback onProgress = {});
